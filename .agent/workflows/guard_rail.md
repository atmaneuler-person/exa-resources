# 🤖 AI 작업 대원칙 (Guard Rails)

이 프로젝트에서 AI는 아래 원칙을 **절대적으로 준수**하며 작업해야 합니다. 이 규칙은 세션 리로드나 모델 변경 시에도 최우선으로 적용됩니다.

## 1. 데이터 보존 및 승인 원칙

- **임의 삭제 금지**: 사용자가 입력한 데이터를 삭제하는 명령어(`db:drop`, `db:seed`, `destroy_all` 등)를 독단적으로 실행하지 않는다.
- **사전 승인 필수**: 데이터 손실이 예상되거나 환경 설정을 크게 바꾸는 작업은 반드시 한국어로 상세히 설명하고 사용자에게 **"YES"** 또는 승인 답변을 받은 후 실행한다.

## 2. 소스코드 로직 보호 원칙

- **로직 분석 우선**: 기존 코드를 수정하기 전, 해당 로직의 설계 의도와 핵심 계산 방식(예: SWV, Bayesian inference 등)을 먼저 분석하여 사용자에게 보고한다.
- **임의 변경 금지**: AI의 판단으로 로직을 '일반화'하거나 '단순화'하여 덮어쓰지 않는다.
- **변경 사항 고지**: 코드를 수정할 때는 정확히 어느 부분이 왜 바뀌는지 사전 설명하고 승인을 얻는다.

## 3. 안전장치 활용

- **Git 활용**: 중대한 수정 전에는 현재 상태를 `Commit`하여 언제든 복구 가능한 상태를 유지한다.
- **백업 활성화**: DB 변경 작업 전에는 `storage/development.sqlite3` 파일을 수동으로 백업한다.

## 4. 커뮤니케이션

- 모든 답변과 설명은 **한국어**로 진행하며, 사용자님의 피드백을 최우선으로 반영한다.
